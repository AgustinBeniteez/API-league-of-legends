Introduction

This is a robust and easy-to-use API built with Node.js and Express that provides up-to-date information about League of Legends (LoL) and Teamfight Tactics (TFT) champions.

Real-Time

Automatic synchronization with the latest patch version from Data Dragon (Riot Games).

Friendly Format

Data transformed into clean and easy-to-iterate arrays for any frontend.

Why use this API?

  • Privacy: We do not collect user data.
  • Speed: Optimized server with low latency.
  • Open Source: Anyone can audit or contribute to the project.

Usage Guide

Implementing the API in your project is simple. You can use native JavaScript fetch or any HTTP client.

How to make a request

To get data for a specific champion, like Lux, simply perform a GET request to the corresponding endpoint:

fetch('https://api-league-of-legends-ruby.vercel.app/lol/champion/Lux')
  .then(response => response.json())
  .then(data => {
    console.log("Name:", data.name);
    console.log("Title:", data.title);
  });

Advanced Usage Examples

1. Getting Champion Splash Arts (Skins)

Each champion object contains a skins array. You can use the num property to build the splash art URL.

// Get the splash art for the second skin
const secondSkin = data.skins[1]; 
const imageUrl = `https://ddragon.leagueoflegends.com/cdn/img/champion/splash/${data.id}_${secondSkin.num}.jpg`;

2. Accessing Detailed Stats

Base stats include health, resource type, and growth values per level.

console.log(`HP: ${data.stats.hp} (+${data.stats.hpperlevel} per level)`);
console.log(`Armor: ${data.stats.armor}`);
console.log(`Resource: ${data.partype}`); // e.g., "Mana", "Energy"

3. Ability Costs and Cooldowns

The spells array contains your Q, W, E, and R skills with all technical meta-data.

const ultimate = data.spells[3]; // The R ability
console.log(`Name: ${ultimate.name}`);
console.log(`Cooldown: ${ultimate.cooldown}s`);
console.log(`Cost: ${ultimate.cost} ${ultimate.costType}`);

Iterating over the JSON response

When requesting the full list of champions, you receive an array of basic champion info. Use this for lists of thumbnails or search filters.

// Example: Filter only Tanks
const tanks = allChampions.filter(c => c.tags.includes('Tank'));

LoL Endpoints

Main endpoints to access the League of Legends database. The base URL is https://api-league-of-legends-ruby.vercel.app.

GET https://api-league-of-legends-ruby.vercel.app/lol/champions

Gets all champions with stats and images.

GET https://api-league-of-legends-ruby.vercel.app/lol/champion/:id

Detail of a champion by name or ID (e.g., /lol/champion/Lux).

GET https://api-league-of-legends-ruby.vercel.app/lol/items

Full list of all game items.

Summoner Spells

Endpoints to get information about summoner spells.

GET https://api-league-of-legends-ruby.vercel.app/lol/spells

Gets all spells available in the game.

GET https://api-league-of-legends-ruby.vercel.app/lol/spell/:id

Detail of a spell by ID or name (e.g., /lol/spell/SummonerFlash).

TFT Endpoints

Specific data for Teamfight Tactics (Current Set).

GET https://api-league-of-legends-ruby.vercel.app/tft/champions

Full list of champions, costs, and tiers.

GET https://api-league-of-legends-ruby.vercel.app/tft/champion/:id

Specific detail of a TFT piece.

Contribution

If you want to contribute to development or run the project locally, follow these steps.

1. Clone the repository

git clone https://github.com/AgustinBeniteez/API-league-of-legends.git
cd API-league-of-legends

2. Install dependencies

npm install

3. Run the server

node index.js

The server will start by default at http://localhost:3000 for local testing.
